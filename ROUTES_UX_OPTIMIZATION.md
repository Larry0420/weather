# Routes Widget UX 優化

## 問題描述
原本的 Routes 頁面存在 UX 問題：
- 用戶一進入頁面就會看到預設的勾選狀態（tick）
- 系統自動選擇第一個地區和第一條路線（index 0）
- 這會讓用戶感到困惑，不知道是否需要點擊

## 優化方案

### 1. 移除預設選擇
- 將 `RoutesCatalogProvider` 中的 `_districtIndex` 和 `_routeIndex` 改為可空類型（`int?`）
- 初始化時不自動選擇任何地區或路線
- 用戶需要主動選擇才能看到勾選狀態

### 2. 改進選擇邏輯
```dart
// 修改前
int _districtIndex = 0;
int _routeIndex = 0;

// 修改後
int? _districtIndex; // 改為可空，表示未選擇
int? _routeIndex; // 改為可空，表示未選擇
```

### 3. 更新顯示邏輯
- 地區選擇器：只有當用戶選擇了地區時才顯示勾選狀態
- 路線選擇器：只有當選擇了地區後才顯示，且沒有預設勾選
- 主內容區域：根據選擇狀態顯示相應的引導信息

### 4. 簡化用戶界面
- 移除所有引導文字和圖標，讓界面更簡潔
- 當沒有選擇時顯示空白區域，不干擾用戶
- 保持選擇器的清晰性，讓用戶自然理解操作流程

### 5. 保持狀態持久化
- 仍然支持保存用戶的選擇到 SharedPreferences
- 但只在用戶主動選擇時才保存
- 當用戶重新選擇地區時，會重置路線選擇

## 優化效果

### 改進前
- ❌ 用戶進入頁面看到預設勾選，感到困惑
- ❌ 不清楚是否需要點擊
- ❌ UX 不夠直觀

### 改進後
- ✅ 清晰的選擇流程：地區 → 路線 → 內容
- ✅ 沒有預設勾選，用戶需要主動選擇
- ✅ 簡潔的界面，無多餘的引導文字
- ✅ 符合用戶預期的交互模式

## 技術實現

### 關鍵變更
1. **RoutesCatalogProvider**：
   - 將索引改為可空類型
   - 更新選擇邏輯和狀態管理

2. **_RoutesPage**：
   - 更新顯示邏輯
   - 改進用戶引導界面
   - 優化選擇器狀態處理

3. **狀態持久化**：
   - 保持選擇的持久化功能
   - 改進狀態重置邏輯

```dart
  // 顯示路線選擇器
else if (district != null)
  // 顯示無路線提示
else
  // 顯示空白區域（無引導文字）
```

## 總結
這次優化解決了 Routes widget 的 UX 問題，讓用戶體驗更加直觀和友好。用戶現在需要主動選擇，而不是被預設選擇所困惑，這符合現代 UI/UX 的最佳實踐。
