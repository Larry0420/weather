# 橫向模式佈局優化總結

## 概述
本次優化針對輕鐵班次應用程式的橫向模式進行了全面的佈局改進，提升了在平板和橫向手機上的使用體驗。

## 主要優化內容

### 1. 車站選擇器優化 (`_OptimizedStationSelector`)

#### 橫向模式特點：
- **緊湊佈局**：減少按鈕和圖標的尺寸，優化間距
- **並排顯示**：左側顯示搜索框和最近使用，右側顯示車站列表
- **響應式設計**：根據螢幕方向自動調整佈局

#### 具體改進：
- 主選擇器按鈕：橫向模式下圖標從 48x48 縮小到 40x40
- 搜索框：橫向模式下使用更小的圓角和內邊距
- 最近使用：橫向模式下只顯示前3個車站，使用更緊湊的間距
- 車站列表：橫向模式下使用更小的列表項目和字體

### 2. 路線選擇器優化 (`_RoutesPage`)

#### 橫向模式特點：
- **並排佈局**：地區選擇器和路線選擇器並排顯示
- **緊湊間距**：減少選項之間的間距，提高空間利用率
- **自適應佈局**：根據螢幕方向自動選擇最佳佈局

#### 具體改進：
- 地區選擇器：橫向模式下使用更小的間距 (3px vs 4px)
- 路線選擇器：橫向模式下使用更緊湊的內邊距
- 選項標籤：橫向模式下使用更小的字體和間距

### 3. 設定頁面優化 (`_SettingsPage`)

#### 橫向模式特點：
- **雙欄佈局**：左側顯示輔助功能設定，右側顯示主題設定
- **網格佈局**：更好地利用橫向空間
- **分組顯示**：相關設定項目分組顯示

#### 具體改進：
- 輔助功能：文字大小、圖示大小、螢幕旋轉設定集中在左側
- 主題設定：主題顏色、深色模式、系統主題設定集中在右側
- 直向模式：保持原有的單欄佈局

### 4. 班次表優化

#### 橫向模式特點：
- **卡片佈局**：優化卡片在橫向模式下的顯示
- **文字溢出處理**：確保長文字在橫向模式下正確顯示
- **響應式間距**：根據螢幕方向調整間距

## 技術實現

### 1. 方向檢測
```dart
final isLandscape = MediaQuery.of(context).orientation == Orientation.landscape;
```

### 2. 條件渲染
```dart
if (isLandscape)
  // 橫向模式佈局
else
  // 直向模式佈局
```

### 3. 響應式尺寸
```dart
width: isLandscape ? 40 : 48,
height: isLandscape ? 40 : 48,
fontSize: (isLandscape ? 14 : 16) * accessibility.textScale,
```

### 4. 自適應間距
```dart
padding: EdgeInsets.all(isLandscape ? 12 : 16),
spacing: isLandscape ? 3.0 : 4.0,
```

## 用戶體驗改進

### 1. 空間利用率
- 橫向模式下更好地利用螢幕寬度
- 減少不必要的空白區域
- 提高信息密度

### 2. 操作便利性
- 並排佈局減少滾動需求
- 相關功能分組顯示，提高查找效率
- 保持觸控目標的合適大小

### 3. 視覺一致性
- 保持與直向模式的視覺一致性
- 使用相同的設計語言和色彩方案
- 平滑的佈局轉換動畫
